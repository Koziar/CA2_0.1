Error Responses:

We use two HTTP status codes. 500 for Internal Server Errors, and 404 for person not found, company not found, and for all others. That way we have our bases covered.

When we get an exception, we map it, and our exception mappers can then convert it into an HTTP responds.

In our mappers in the exception package, you can see how we use Gson to map an exception to Json. This is made possible with the help of our ErrorMessage.java class.
